# 华中10型数控系统开放平台二次开发指南
**版本**：V2
**日期**：20260324

# 1. 概述与开发基础
为快速适配多样化市场应用场景，华中数控扩展二次开发平台，完善开发流程与体系，提供开发环境、工具及典型案例，形成面向不同应用场景、开发群体、开发层次（应用层/内核层/硬件接口层）、代码深度（无代码/低代码/高代码）的二次开发平台，实现系统厂商软件与用户二次开发软件独立开发、升级、维护，为数控系统与行业深度结合提供技术支撑。

## 1.1 开发类型介绍
华中数控系统包含808系列、818系列、848系列、华中10型，满足高中低端机床配套需求。系统提供开放式软硬件平台，支持定制化开发，包括定制操作面板、HMI界面、加工工艺、智能化功能集成、数字化车间/产线系统等。

### 1.1.1 定制化数控硬件面板
针对机床厂家操作方式差异需求，提供操作面板定制方案：
- 标准面板：华中数控系统848系列标准操作面板
- 定制面板：用户自定义布局与操作方式

### 1.1.2 定制化数控软件界面
提供三层界面集成方案：
1. **简单自定义控件**：不改变布局风格，新增参数设置、程序导入控件，使用组态工具快速实现
2. **加工工艺集成界面**：支持布局与风格定制，基于QT开发包实现个性化界面
3. **独立上位机软件**：支持复杂工艺与功能集成（CAM、传感器监控），兼容Linux/Windows，支持第三方IPC与全触屏

### 1.1.3 智能化功能应用
支持机床健康监控、刀具诊断、工艺优化、自适应参数、热误差补偿、轮廓误差补偿等，提供指令域大数据接口与传感器接入接口，可开发智能APP部署于系统或云端。

构建**端-边-云**智能体系架构，支持数字化车间看板、轻量化MES、DNC、MDC等应用。

### 开发分类（按运行位置）
1. **数控装置端**：定制界面/功能、智能APP、硬件面板定制
2. **边缘端**：基于inc_cloud平台与iNC-Box边缘设备开发
3. **云端**：基于数据传输协议的远程数据应用开发

## 1.2 开发平台及环境
### 1.2.1 硬件平台环境
华中10型硬件组成：
- 触显面板：24"/17"全触屏，支持横竖屏与第三方屏
- MCP面板：简洁版，支持虚拟NC键盘
- 上位机IPC220B：11代i7，16G内存/256G硬盘，Win10，支持AI芯片
- 下位机IPC220B：同配置，定制Linux+实时补丁，支持NCUC1.0/2.0、EtherCAT
- 智能传感模块：NCUC2.0，模拟量采集，支持振动/功率传感器，AI芯片≥18TOPS
- 120VG/VG-PRO伺服驱动：8k精插补，高精度数据采集
- HIO-1000高速I/O模块：NCUC1.0/2.0，高速I/O，温度传感器接入

### 1.2.2 软件平台环境
分层设计：
- **数控核心层**：译码、插补、轴/通道控制，多轴多通道、RTCP、误差补偿
- **数控应用业务层**：基础HMI、一键报表、三维仿真、测量标定、伺服调试
- **智能化应用层**：刀具检测、工艺优化等智能应用

提供集成开发环境、运动规划、HMI开发、智能感知、大数据、智能化接口。

#### 2. 环境搭建步骤
- 文件解压至**全英文无空格**路径
- Windows：开发调试；Linux：编译生成装置可执行文件
- 安装要求：VS/QT/VMware默认路径，禁止修改

**VS2010安装**
1. 运行autorun.exe → 安装VS2010
2. 接受许可 → 仅勾选Visual C++ → 安装
3. 完成后退出后续组件提示

**QT安装**
1. 官网下载：archive/qt/4.8/4.8.7 或 archive/qt creator/3.2/3.2.2
2. 注册登录 → 默认路径 → 安装组件 → 完成

**QT环境配置**
1. 将qmake路径加入系统环境变量
2. Qt Creator → 工具 → 选项 → 设置编码与构建套件
3. 添加QT4.8.7 → 应用保存

**VMware安装**
官网下载，默认下一步安装即可。

#### 华中10型环境配置
**Windows**
1. 解压开发包，添加mosquitto至环境变量
2. 设置VMnet8 IP（与下位机同网段，如上位机192.168.1.100，下位机192.168.1.59）
3. 关闭防火墙，开启SMB 1.0/CIFS共享

**下位机虚拟机**
1. 打开hnc_Linux5.15.01/02镜像
2. 网络设为NAT，配置共享文件夹（全英文路径）
3. ifconfig设置IP，与上位机同网段

**NC-Link配置**
修改`nclink.cfg`、`ncsync2-upper.xml`、`ncsync2-lower.xml`中server_ip为上位机IP

**上下位机启动顺序**
1. 启动下位机虚拟机 → 运行下位机软件
2. 启动mosquitto → 启动ncsync2-upper → 启动上位机界面

**注意事项**
- 双屏扩展：主屏左，扩展屏右
- 无法启动：检查网络、SMB共享、IP配置

### 1.2.4 测试验证环境构建
1. **模拟测试**：使用系统模拟器，虚拟数据调试
2. **实验平台**：实物数控系统+伺服+电机，实测数据验证
3. **机床平台**：整机+传感器，现场功能验证

**编译与升级**
1. 虚拟机编译：`qmake` → `make clean` → `make`
2. 打包：7zip压缩为tar，改后缀`.BTF`，生成MD5
3. 升级：U盘导入系统，重启生效
